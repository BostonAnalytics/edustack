9 Administering Coursework with GitHub Classroom

As you transition to a reproducible, version-controlled authoring workflow using Quarto and GitHub, managing student assignments becomes the next logical challenge. Traditional Learning Management Systems (LMS) like Blackboard are excellent for delivering content, but they can be cumbersome when distributing code templates, grading programming assignments, or managing student collaboration.

This is where GitHub Classroom shines. GitHub Classroom is a teaching tool designed to bridge the gap between professional version control and academic assignment management. By integrating GitHub Classroom into your course workflow, you can automate repository creation, distribute starter code seamlessly, and even implement autograding.

9.1 Core Features for Educators

GitHub Classroom offers several powerful features that simplify the teaching and learning experience, making it the perfect companion for a Quarto-based course.

9.1.1 Template Repositories

The Quarto framework we established earlier involves generating a specific directory structure (M01, M02) with predefined lecture notes, labs, assignments, projects, presentations, and participation files (M01_A.qmd, M01_Proj.qmd, M01_Lab1.qmd). With GitHub Classroom, you can turn these generated directories into template repositories.

When you create an assignment, you can link it to your template repository. GitHub Classroom will then automatically generate a private, boilerplate repository for every student (or group) who accepts the assignment. This ensures every student starts with the exact same file structure and Quarto configurations.

9.1.2 Autograding

For courses that involve programming or reproducible data analysis, grading can be incredibly time-consuming. GitHub Classroom allows you to configure automated tests that run every time a student pushes a commit to their assignment repository.

Using GitHub Actions under the hood, these autograding scripts can check for syntax errors, verify that a Quarto document compiles correctly, or validate the output of a data analysis script.

9.1.3 Feedback Pull Requests

Instead of downloading zipped folders of student code from Blackboard, GitHub Classroom can automatically open a Feedback Pull Request in every student’s assignment repository. This allows you, or your teaching assistants, to leave line-by-line comments directly on the student’s code, mimicking a professional code review process.

9.1.4 LMS Integration

While we are using Blackboard Ultra as the primary content delivery platform, GitHub Classroom can connect directly to your LMS. By linking your Blackboard course to your GitHub Classroom, you can import your student roster directly. This ensures that every GitHub account is mapped to a specific student ID, preventing the common headache of figuring out which anonymous GitHub handle belongs to which student.

9.2 Integrating Classroom with Your Quarto Workflow

To implement GitHub Classroom effectively in this framework, consider the following workflow for each module:

  1. Generate the Module: Use scripts/generate_modules.py to create the M01 folder and its M01_Lab1.qmd, M01_A.qmd, or M01_Proj.qmd files.
  2. Author the Assignment: Write the assignment instructions and provide any starter data inside the data/ folder.
  3. Create the Template: Push this module folder to a dedicated GitHub repository and mark it as a “Template Repository” in the repository settings.
  4. Publish the Assignment: In GitHub Classroom, create a new assignment, select your template repository, and generate an invitation link.
  5. Post to Blackboard: Finally, take the GitHub Classroom invitation link and embed it directly into your Blackboard Ultra learning module.

When students click the link in Blackboard, they will be taken to GitHub Classroom, where their personal, version-controlled Quarto workspace will be instantly generated. This workflow completely eliminates the friction of file distribution and empowers students to work in the same professional environments they will use in their careers.

9.3 Integrating GitHub Classroom in an Analytics Class

In an analytics class, GitHub Classroom can serve as the distribution and collection layer for coding work. It can be used to create private repositories for each student or group, share starter Quarto, Python, or notebook files, and keep the assignment structure consistent across sections.

It also makes feedback easier. Students submit work through GitHub, and the instructor can review commits, compare revisions, and leave comments directly on files instead of managing multiple uploaded document versions through the LMS.
