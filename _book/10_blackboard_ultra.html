10  Blackboard Ultra: Deploying Quarto Content to Your Course Shell

10.1 Introduction

Blackboard Ultra is Boston University’s learning management system. It serves as the hub where students access your course materials, submit assignments, engage in discussions, and check their grades. The power of Blackboard Ultra emerges when you integrate it with your professionally-designed course materials created in Quarto.

This chapter guides you through the strategic integration of Quarto-generated course content with Blackboard Ultra’s content collection system. Rather than manually creating and updating content in Blackboard, you’ll develop your materials once in Quarto, generate HTML files, and deploy them as a complete package into Blackboard—making updates seamless and maintaining a single source of truth for your course materials.

By the end of this chapter, you’ll understand: - How Blackboard Ultra manages content and links to external resources - The workflow for packaging Quarto content into Blackboard-ready formats - The initial setup process with BU’s Blackboard team - The specific folder structure that enables seamless content management - How to deploy updates throughout the semester without disrupting your course

10.2.1 Courses: Your Course Shell

Your Blackboard course is the container for everything—all materials, assessments, discussions, and grades. Within Blackboard, you organize content into logical sections called modules.

10.2.2 Modules: Organizing Content into Units

Modules correspond to weeks, units, or thematic sections of your course. Each module typically contains lectures, readings, assignments, and discussions related to a specific topic.

10.2.3 Content Collection: The File System

Blackboard maintains a content collection—essentially a cloud file system where all your files are stored. Rather than embedding content directly in Blackboard’s interface, you upload files (HTML, PDFs, images) to the content collection and then link to them from various course elements.

This is where your Quarto-generated HTML files live. Once uploaded to the content collection, Blackboard displays them through its interface, maintaining consistent branding and navigation.

10.3 The Strategic Advantage: Quarto + Blackboard Integration

Rather than creating content in Word or directly in Blackboard’s text editor, you’ll:

  1. Author in Quarto: Create your course materials in clean, maintainable Quarto files (.qmd)
  2. Generate HTML: Export everything as a complete website with quarto render
  3. Package as _site.zip: Create a deployment-ready archive
  4. Upload to Blackboard: Use Blackboard’s content collection to import the entire package
  5. Link from Modules: Create course module structures that point to your HTML content
  6. Update Seamlessly: When content changes, regenerate _site.zip and re-upload—Blackboard replaces old files with new versions

10.3.1 Why This Approach?

Single Source of Truth: Your course materials exist in one place (your Quarto project). Both you and your students work from the same, authoritative version.

Professional Presentation: Quarto produces clean, accessible HTML with consistent formatting. Your course looks polished without spending time on Blackboard’s styling.

Version Control: Your Quarto files live in Git, creating a complete history of how your course evolved.

Batch Updates: Fix typos, update examples, or revise content, then deploy all changes at once—not one file at a time.

Consistency Across Semesters: Template your course structure, then swap in new content for each term.

Purpose: Provides a module overview, learning objectives, and synthesis of concepts.

Content Examples: - Summary of key concepts - Module learning outcomes - Prerequisites - Recommended reading order - Time estimates for completion

In Quarto: Create this as a dedicated .qmd file that renders as your module introduction. For example:

## Learning Objectives

By the end of this module, you will:
- Understand principles of effective data visualization
- Create interactive visualizations with D3.js
- Apply color theory and accessibility standards

## Module Overview

This module explores how to transform raw data into compelling visual narratives...

In Blackboard: Link this as the first item in each module. Students see it immediately upon entering, setting context and expectations.

10.5.2.3.2 Presentation

Format: The slide presentation from your live lecture or a structured presentation of key concepts.

In Quarto: Either: - Export slides as static HTML for viewing in the browser - Include links to PDF versions of presentations - Create interactive HTML presentations using Quarto’s reveal.js integration

Tip: Always provide both presentation and notes. Some students learn better from visuals; others prefer reading detailed explanations.

10.6.2 Configuration: _quarto.yml

Set up your _quarto.yml to generate a website structure compatible with Blackboard:

project:
  type: website
  output-dir: _site
  render:
    - "*.qmd"
    - "*/**.qmd"

website:
  title: "Your Course Title"
  site-url: "https://pages.github.com/your-course"
  repo-url: "https://github.com/your-org/your-course"
  repo-actions: [edit, issue, source]
  
  navbar:
    background: primary
    search: true
    left:
      - href: index.qmd
        text: Home
      - text: Course Modules
        menu:
          - href: module-01/wrap-up.qmd
            text: "Module 1"
          - href: module-02/wrap-up.qmd
            text: "Module 2"
          - text: "---"
          - text: Resources
          - href: resources.qmd
            text: Course Resources

format:
  html:
    theme: cosmo
    css: custom.css
    toc: true
    toc-depth: 3
    code-fold: false
    code-copy: true

zip -r _site.zip _site/

10.6.4 Image References: Why They Persist

One important note about image links: When you reference images in your Quarto files and they’re included in _site.zip, Blackboard will import them along with the HTML. This means your image links will continue to work because:

  1. Relative paths: If your HTML files reference images with relative paths (e.g., ../images/diagram.png), the image hierarchy is preserved
  2. Content collection structure: Blackboard maintains the folder structure when you import _site.zip
  3. Automatic linking: Blackboard’s content collection system updates internal references automatically

Important: Always use relative paths in your Quarto files, not absolute URLs:

❌ Don’t do this:

![Diagram](https://example.com/images/diagram.png)

✓ Do this:

10.8 Best Practices for Quarto + Blackboard Integration

10.8.1 File Naming Conventions

Use clear, consistent naming that makes sense in both Quarto file systems and Blackboard:

✓ Good: - module-01-intro.qmd - lecture-1-foundations.qmd - assignment-project-proposal.qmd - lab-01-sql-basics.qmd

❌ Avoid: - module1.qmd - lecture.qmd - assignment.qmd - lab.qmd

10.8.2 URL/Link Strategy

When creating cross-links in your course materials:

Linking within Blackboard: Use relative paths that Blackboard can resolve:

See [this tutorial](../lecture-1/tutorial.html)

Linking to external resources: Use full URLs:

Check out [this guide](https://example.com/guide)

Linking to assignments: Reference Blackboard assignment links:

Submit your work via [this Blackboard assignment link](https://blackboard.bu.edu/webapps/assignment/...)

10.8.3 Maintenance During the Semester

Content Updates Without Redeployment: - Small text corrections: Fix in Quarto, render, and re-upload _site.zip - Broken links: Test regularly, fix in Quarto, re-render - Typos in assignments: Fix immediately and update within 24 hours

When NOT to Re-upload _site.zip: - After students have already submitted work on an assignment (the submission date might change if Blackboard refreshes) - If you’re only updating one file (upload it individually instead) - During peak submission periods (upload during off-hours to avoid conflicts)

Communication with Students: - If you make significant content changes, post an announcement: “Updated Module 3 lecture notes—please review the latest version” - Include version numbers in assignment prompts: “Submit your response to Assignment 3 (v1.1)”

10.8.4 Disaster Recovery and Rollback

Because your Quarto files are in version control (Git):

  1. Keep previous deployments: Don’t delete old _site.zip files

  2. Tag stable versions: In Git, tag releases after successful deployments

    git tag -a v1.0-spring-2026 -m "Spring 2026 course - first deployment"

  3. Rollback if needed: If something breaks, restore from the previous version:

    git checkout v1.0-spring-2026
quarto render
Compress-Archive -Path _site -DestinationPath _site.zip
# Upload the previous version to Blackboard

10.11 Troubleshooting Common Issues

10.11.1 Problem: Links Break After Uploading to Blackboard

Cause: Absolute URLs in Quarto files or incorrect relative paths

Solution: - Check all image and link references in your .qmd files - Use relative paths: ../images/logo.png not /images/logo.png - Update any absolute URLs to relative paths - Re-render and re-upload

10.11.2 Problem: Images Don’t Display in Blackboard

Cause: Images not included in _site/ folder or path references are wrong

Solution: - Ensure images are in a subfolder of your Quarto project (e.g., images/) - Verify Quarto rendering includes images in _site/ - Check that _site.zip includes image files - Verify relative paths in HTML match actual image locations

10.11.3 Problem: Content Collection Upload Fails

Cause: File size too large or connection interrupted

Solution: - If _site.zip is very large (>1GB), compress images bash # Reduce image quality in Quarto with CSS img { max-width: 100%; height: auto; } - Try uploading individual files instead of _site.zip - Contact Andy if technical issues persist

10.11.4 Problem: Students See Old Content After Re-upload

Cause: Browser caching

Solution: - Instruct students to clear browser cache or use Ctrl+Shift+Delete - Use browser-specific cache-busting in _quarto.yml - Announce updates in course announcements

10.11.5 Problem: Module Structure Changes Required Mid-Semester

Cause: Didn’t lock structure with Andy before launch

Solution: - Contact Andy before making structural changes - Update both Quarto and Blackboard simultaneously - Communicate changes to students - Test thoroughly before making live
